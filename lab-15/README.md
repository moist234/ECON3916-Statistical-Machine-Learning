The Polynomial Trap: Bias-Variance Tradeoff
This lab translates one of machine learning's core theoretical tensions into hands-on empirical analysis. Using synthetic sine-wave data (n=50 training, n=200 test), I fit polynomial regression models from degree 1 through 15, demonstrating how training error monotonically decreases while test error forms a U-shape. The complexity curve identified degrees 3 to 5 as optimal, consistent with the known data-generating process.
I then implemented K-fold cross-validation from scratch, partitioning training data into five rotating folds to estimate out-of-sample error without touching the holdout set. The CV-selected degree matched the true test-optimal degree, validating cross-validation as a reliable model selection tool in practice.
The lab concludes with a real-data challenge using the Ames Housing dataset (1,460 observations, 80 features). A kitchen-sink model using all numeric features achieved higher training R-squared but worse CV RMSE than a parsimonious 5-feature model built on correlation-ranked predictors. This result illustrates how high-dimensional models overfit even at modest feature counts relative to sample size.
Tools: Python, NumPy, scikit-learn (PolynomialFeatures, LinearRegression, cross_val_score), and Matplotlib.
This project reflects my ability to connect statistical theory to implementation and communicate model selection decisions with empirical rigor.
